# Support

Thank you for using MDW.

This document explains how to get help, report issues, and request new features.

---

# Documentation

Before requesting support, please review the available documentation.

Useful resources include:

* README.md
* README.tr.md
* CHANGELOG.md
* SECURITY.md
* CONTRIBUTING.md
* Documentation in the `docs` directory

Many common questions are answered there.

---

# Reporting Bugs

If you believe you have found a bug, please open a GitHub Issue.

Include the following information whenever possible:

* MDW version
* PowerShell version
* Windows version
* Command executed
* Expected behavior
* Actual behavior
* Error message
* Steps to reproduce
* Relevant logs or screenshots

Providing complete information helps identify and resolve issues more quickly.

---

# Feature Requests

Suggestions for improvements are welcome.

Before submitting a feature request:

* Check whether a similar request already exists.
* Clearly explain the problem you are trying to solve.
* Describe the proposed solution.
* Include any additional context if relevant.

Please note that new features are evaluated based on the project's goals and roadmap.

---

# Security Issues

Do **not** report security vulnerabilities through public GitHub Issues.

Please follow the process described in **SECURITY.md**.

---

# Discussions

General questions, ideas, and project discussions should be shared through GitHub Discussions (if enabled) or GitHub Issues.

---

# Support Scope

Support is provided on a best-effort basis.

Priority is generally given to:

* Reproducible bugs
* Documentation issues
* Build and release problems
* Compatibility issues
* Security-related concerns

---

# Response Expectations

Response times may vary depending on the complexity of the request and current project activity.

While every effort is made to review reports promptly, response times cannot be guaranteed.

---

# Thank You

Your feedback, bug reports, documentation improvements, and feature suggestions help improve MDW for everyone.

Thank you for supporting the project.
