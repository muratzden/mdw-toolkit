# Security Policy

## Supported Versions

Security updates are provided only for supported stable releases.

| Version | Supported |
| ------- | :-------: |
| 1.x.x   |     ✅     |
| 0.x.x   |     ❌     |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public GitHub Issue**.

Instead, report the issue privately so it can be investigated and resolved before public disclosure.

Please include the following information:

* MDW version
* PowerShell version
* Windows version
* Description of the vulnerability
* Steps to reproduce
* Expected behavior
* Actual behavior
* Proof of concept (if available)
* Any relevant logs or screenshots

---

## Response Process

Every security report will be:

1. Acknowledged.
2. Investigated.
3. Validated.
4. Fixed if confirmed.
5. Included in a future security release.

Our goal is to acknowledge valid reports within **7 business days**.

---

## Responsible Disclosure

Please allow reasonable time for a fix before publicly disclosing a security issue.

Responsible disclosure helps protect all users of the project.

---

## Security Best Practices

To help keep your environment secure:

* Use the latest stable version of MDW.
* Keep PowerShell up to date.
* Keep Git up to date.
* Use trusted plugin sources.
* Review generated release packages before distribution.
* Do not execute untrusted scripts inside your workspace.

---

## Contact

Security reports should be submitted through the project's private support channel.

Please do not disclose security vulnerabilities publicly until they have been investigated and resolved.
